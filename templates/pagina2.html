Encabezados HTTP: La Comunicación Invisible de la Web

¿Qué son los Encabezados HTTP?

Los encabezados HTTP son campos de metadatos que se envían junto con las solicitudes y respuestas HTTP. Proporcionan información adicional sobre la transacción que está teniendo lugar entre el cliente y el servidor. Estos encabezados son invisibles para el usuario final pero son fundamentales para el correcto funcionamiento de la comunicación web.

Estructura de los Encabezados

Los encabezados HTTP siguen una estructura simple:

Nombre-del-Encabezado: Valor

Por ejemplo: Content-Type: text/html; charset=UTF-8

Tipos de Encabezados HTTP

Encabezados de Solicitud (Request Headers)

Enviados por el cliente (navegador) al servidor:

Accept

Propósito: Especifica los tipos de contenido que el cliente puede procesar

Ejemplo: Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8

Uso: El servidor utiliza esta información para determinar qué formato de respuesta enviar

User-Agent

Propósito: Identifica el navegador y sistema operativo del cliente

Ejemplo: User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36

Uso: Permite al servidor adaptar el contenido según el cliente (diseño responsivo, compatibilidad)

Authorization

Propósito: Contiene las credenciales para autenticar al cliente

Ejemplo: Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Uso: Autenticación en APIs y recursos protegidos

Cookie

Propósito: Envía cookies almacenadas previamente por el servidor

Ejemplo: Cookie: session_id=abc123; theme=dark

Uso: Mantener estado de sesión, preferencias del usuario

Encabezados de Respuesta (Response Headers)

Enviados por el servidor al cliente:

Server

Propósito: Identifica el software del servidor web

Ejemplos:

  • Server: Apache/2.4.41 (Ubuntu)
  • Server: nginx/1.18.0
  • Server: Microsoft-IIS/10.0
  • Server: cloudflare

Uso: Información para debugging, estadísticas de mercado

Date

Propósito: Indica la fecha y hora cuando se generó la respuesta

Ejemplo: Date: Wed, 21 Oct 2015 07:28:00 GMT

Formato: Siempre en GMT (Greenwich Mean Time)

Uso: Cacheo, logging, sincronización

Content-Type

Propósito: Especifica el tipo de contenido y codificación

Ejemplos:

  • Content-Type: text/html; charset=UTF-8 - Página HTML
  • Content-Type: application/json - Datos JSON
  • Content-Type: image/png - Imagen PNG
  • Content-Type: text/css - Hoja de estilos CSS
  • Content-Type: application/javascript - Código JavaScript

Uso: El navegador usa esta información para saber cómo procesar el contenido

Content-Length

Propósito: Indica el tamaño del cuerpo de la respuesta en bytes

Ejemplo: Content-Length: 1234

Uso: Permite al cliente saber cuántos datos esperar

Set-Cookie

Propósito: Envía una cookie al cliente para que la almacene

Ejemplo: Set-Cookie: session_id=abc123; Path=/; HttpOnly; Secure

Atributos importantes:

  • HttpOnly: La cookie no es accesible desde JavaScript
  • Secure: Solo se envía sobre HTTPS
  • SameSite: Controla cuándo se envía la cookie

Encabezados de Seguridad

Content-Security-Policy (CSP)

Propósito: Define qué recursos puede cargar la página

Ejemplo: Content-Security-Policy: default-src 'self'; script-src 'self' https://apis.google.com

Beneficio: Previene ataques XSS (Cross-Site Scripting)

X-Frame-Options

Propósito: Controla si la página puede ser mostrada en un iframe

Valores:

  • DENY - Nunca en iframe
  • SAMEORIGIN - Solo en iframe del mismo origen
  • ALLOW-FROM uri - Solo desde URI específica

Beneficio: Previene ataques clickjacking

Strict-Transport-Security (HSTS)

Propósito: Fuerza el uso de HTTPS

Ejemplo: Strict-Transport-Security: max-age=31536000; includeSubDomains

Beneficio: Previene ataques man-in-the-middle

Encabezados de Cacheo

Cache-Control

Propósito: Controla cómo y por cuánto tiempo se debe cachear el contenido

Ejemplos:

  • Cache-Control: no-cache - Siempre validar con el servidor
  • Cache-Control: max-age=3600 - Cachear por 1 hora
  • Cache-Control: public - Puede ser cacheado por cualquier caché
  • Cache-Control: private - Solo cachear en el navegador del usuario

ETag

Propósito: Identificador único para una versión específica del recurso

Ejemplo: ETag: "33a64df551425fcc55e4d42a148795d9f25f89d4"

Uso: El cliente puede enviar If-None-Match para verificar si el recurso ha cambiado

Last-Modified

Propósito: Indica cuándo fue modificado por última vez el recurso

Ejemplo: Last-Modified: Wed, 21 Oct 2015 07:28:00 GMT

Uso: Permite cacheo condicional con If-Modified-Since

Encabezados de CORS

Access-Control-Allow-Origin

Propósito: Especifica qué orígenes pueden acceder al recurso

Ejemplos:

  • Access-Control-Allow-Origin: * - Cualquier origen
  • Access-Control-Allow-Origin: https://example.com - Solo este dominio

Access-Control-Allow-Methods

Propósito: Especifica qué métodos HTTP están permitidos

Ejemplo: Access-Control-Allow-Methods: GET, POST, PUT, DELETE

Encabezados de Compresión

Accept-Encoding

Propósito: Indica qué algoritmos de compresión acepta el cliente

Ejemplo: Accept-Encoding: gzip, deflate, br

Content-Encoding

Propósito: Indica qué compresión se aplicó al contenido

Ejemplo: Content-Encoding: gzip

Códigos de Estado HTTP Relacionados

Los encabezados trabajan junto con códigos de estado:

  • 200 OK: Solicitud exitosa
  • 304 Not Modified: Recurso no ha cambiado (usado con encabezados de cacheo)
  • 401 Unauthorized: Requiere autenticación (usado con WWW-Authenticate)
  • 403 Forbidden: Acceso denegado
  • 404 Not Found: Recurso no encontrado
  • 500 Internal Server Error: Error del servidor

Herramientas para Inspeccionar Encabezados

  • Herramientas de desarrollador del navegador: F12 → Network
  • cURL: curl -I https://example.com
  • Postman: Cliente gráfico para APIs
  • HTTPie: Cliente HTTP desde línea de comandos
  • Online tools: Redbot.org, web-sniffer.net

Buenas Prácticas

  • Seguridad: Always use security headers in production
  • Performance: Configure proper caching headers
  • Compression: Enable gzip/brotli compression
  • HTTPS: Always use HTTPS in production
  • Monitoring: Log and monitor important headers

Los encabezados HTTP son el lenguaje invisible que permite que navegadores y servidores se comuniquen eficientemente, proporcionando funcionalidades esenciales como seguridad, cacheo, autenticación y optimización del rendimiento.
